Use `todo_write` to track work when the request spans 3+ steps or multiple files. Skip it for single-step changes, direct answers, or trivial requests — ceremony without signal wastes a turn.

When you do create a todo list, make `content` a short label (5-10 words) and put file paths, implementation steps, and specifics in `details`. Keep exactly one task `in_progress` and mark `completed` immediately when done — batching kills the signal the list was supposed to provide.
